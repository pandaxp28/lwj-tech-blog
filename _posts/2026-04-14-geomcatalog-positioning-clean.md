GeomCatalog の訴求を整理すると、現時点では **PDM と言い切るよりも、3D CADデータ管理ソフトとして表現する方が適切** です。

理由は明確です。GeomCatalog が現在強く価値を出せるのは、**STEP / IGES / STL などの 3D ファイルを、サムネイル付きで整理し、検索し、再利用しやすくすること** にあります。ここを正確に伝えた方が、機能との整合が取りやすく、見込みユーザーにも誤解なく届きます。

## いまの GeomCatalog をどう表現するべきか

製品カテゴリ名として最も無理がないのは、**3D CADデータ管理ソフト** です。

この表現なら、

- STEP / IGES / STL の管理に対応していること
- 設計データの整理と検索が主目的であること
- 一般的な PDM ほど重い運用前提ではないこと

が自然に伝わります。

一方で、**PDMソフト**、**PLM**、**図面管理システム** といった表現は、現状では期待値を上げすぎる可能性があります。

たとえば PDM という言葉からは、版管理、承認フロー、チェックイン / チェックアウト、権限管理、BOM 連携まで想起されやすくなります。GeomCatalog の実態はそこではなく、まず **3D データを探しやすくし、扱いやすくするローカル型の実務ツール** です。

## まず前面に出すべき訴求

現状の訴求として最も強いのは、次の一文です。

**STEP / IGES / STL を、サムネイル付きで整理・検索。**

この文言は、対応形式と価値を短く同時に伝えられます。

GeomCatalog は、3D データを単にフォルダへ保存するだけでなく、**一覧しやすく、見つけやすく、再利用しやすい形へ変える** ことに意味があります。設計現場では、過去データが存在していても「探せない」「見分けにくい」「中身が開かないと分からない」という問題が起こりがちです。

その点で、サムネイル表示は単なる見た目ではありません。**必要なモデルへ速く到達するための実務機能** です。

## ホームページではどう見せるべきか

ホームページの見出しとしては、次の組み合わせが扱いやすい構成です。

- カテゴリ名: **3D CADデータ管理ソフト**
- キャッチコピー: **STEP / IGES / STL を、サムネイル付きで整理・検索。**

さらに、その下に補足説明として次のような 1 文を置くと、製品の立ち位置が明確になります。

> GeomCatalog は、STEP / IGES / STL などの 3D設計データをローカル環境で登録・検索・整理できる Windows 向けソフトです。

これにより、

- 何のソフトか
- どんなファイルに対応するか
- どんな使い方を想定しているか
- Windows 向けであること

が短く伝わります。

## GeomCatalog の強みは「重すぎないこと」でもある

GeomCatalog の価値は、多機能化しすぎていない点にもあります。

本格的な PDM や PLM は、組織運用に乗せたときの強さがありますが、そのぶん導入と定着の負荷も大きくなります。対して GeomCatalog は、まず **現場で散らばった 3D データを見つけやすくする** ところから始められるのが利点です。

つまり、GeomCatalog は「大規模管理システムの代替」を狙うより、**現場の 3D データ整理を前進させる入口** として訴求した方が、製品の強みと一致します。

## 現時点の採用文言

現段階で採用するなら、次の組み合わせが最も安定しています。

- 製品カテゴリ名: **3D CADデータ管理ソフト**
- キャッチコピー: **STEP / IGES / STL を、サムネイル付きで整理・検索。**
- 1文紹介: **GeomCatalog は、3D設計データをローカルで管理し、サムネイル付きで見つけやすくする Windows 向けアプリです。**

GeomCatalog は、3D データを単に保存するだけのツールではなく、**探せる資産に変えるためのソフト** として育てていくのが自然です。

今後は、タグ、コメント、お気に入り、StepGeomChecker 連携といった要素も含めて、3D データの再利用性をどう高めるかをさらに詰めていきます。
