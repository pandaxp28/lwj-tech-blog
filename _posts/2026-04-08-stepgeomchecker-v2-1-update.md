StepGeomChecker V2.1 では、主に**読み込み高速化**、**キャンセル応答の改善**、**メモリ使用の見直し**を進めました。

今回の更新で特に大きいのは、**V2.0系から比べて再読込がかなり高速化したこと**です。見た目の機能追加というよりも、重いモデルを扱うときの待ち時間や扱いやすさを改善する方向が中心です。

## 今回の更新の中心

V2.1 で大きいのは、同じ STEP ファイルを再度開くときの読み込み改善です。

これまでは DirectX 表示経路で `TransferCache` を意図的に使わない構成になっており、キャッシュが保存済みでも毎回フル読込が走りやすい状態でした。V2.1 ではこの見直しを行い、DirectX + STEP でも transfer cache を通常どおり参照するようにしています。cache が使えない場合は、従来どおりフル読込へフォールバックします。

## V2.0系から見た再読込の高速化

性能記録では、`etc.STEP`、DirectX、`deflection=0.5`、Warm 再読込、`TransferCache=hit`、`MeshCache=hit` の条件で、`AnalyzeTotal` は `374931 ms` から `11104 ms`、`Load_Total` は `372376 ms` から `8382 ms` まで短縮されています。

これは `AnalyzeTotal` で約 **97.0 % 短縮 / 約 33.8 倍**、`Load_Total` で約 **97.7 % 短縮 / 約 44.4 倍** に相当します。つまり、V2.0系までの待ち方に比べると、同じファイルをもう一度開くケースでは体感差がかなり大きくなっています。

今回の数値は初回読込ではなく、同じファイルを再度開くケースでの改善効果です。効いている主因は、DirectX でも `TransferCache` を再利用するようにしたことです。

## キャンセルの効き方を改善

大容量 STEP では、`Cancel` を押しても native 側の重い処理が終わるまで戻らず、効いていないように見える場面がありました。

V2.1 では progress callback を「通知専用」ではなく、継続 / 中断を返せる形へ見直し、`Load` や `TessellateParts` の主要区間でキャンセル要求を拾えるようにしています。

また、XDE 読み込みは `Perform(...)` の一発処理をやめ、`ReadFile(...)` と transfer を分離しました。これにより、どこで待っているかが分かりやすくなり、複数 root を持つ大容量 STEP では root 単位でキャンセルが効きやすくなっています。

さらに、`Loading part colors ...` や `Loading face colors ...` の区間でも cancel を確認できるようにし、色取得フェーズに入ったまま長く止まりにくい構成へ寄せています。

表示面でも、`Cancel` 押下後は通常の `Loading ...` 表示を続けるのではなく、**キャンセル待ちであることが分かる文言** に切り替えるようにしました。止まるまで数秒以上かかる場合でも、今どの状態かが見えやすくなっています。

## メモリ使用の見直し

大きなモデルを連続で読むときは、旧モデルの表示メッシュが残ったまま新規読込へ進み、メモリピークが高くなりやすい問題がありました。

V2.1 では、新規モデル読込前に旧表示メッシュを先行解放するようにしています。DirectX ビューでは、選択オーバーレイや計測オーバーレイ、STEP 色、`triPartIds` なども同時にクリアし、切替時の負荷を下げる方向で整理しています。

加えて、DirectX 表示経路で持っていた不要な CPU clone や重複保持も見直しました。packed vertex 配列の CPU clone を持たず、`indices`、`triFaceIds`、`triPartIds`、STEP 色配列も不要な clone を避ける方針にしています。大きなモデルでのメモリピークや GC 負荷の低減が狙いです。

## キャッシュ運用も整理

V2.1 では、`MeshPartsCache` と `TransferShapeCache` がディスク上に蓄積し続ける点にも手を入れています。

総量上限を設け、新規保存後に上限を超えていた場合は古いキャッシュから順に削除するようにしました。初期値は `MeshPartsCache` が **8 GiB**、`TransferShapeCache` が **4 GiB** です。大きいファイルを何度も扱う環境でも、ローカルストレージを圧迫し続けにくくなります。

## V2.1 で狙っていること

今回の V2.1 は、派手な新機能を増やす更新ではありません。

その代わり、重い STEP / IGES を開くときに

- 同じファイルを再度開くときはできるだけ速くする
- 止めたいときは、止めたいことがちゃんと伝わるようにする
- 大きいモデルを切り替えるときのメモリ負荷を減らす

という、日常的な使い勝手の部分を詰めています。

今後も StepGeomChecker は、3D データを軽く開いて、必要な確認をすばやく進められる方向で改善を重ねていきます。
